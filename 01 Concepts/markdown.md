# Markdown

Use this note for Obsidian Markdown conventions that matter for research notes.

## Useful Patterns

- Link notes with double brackets around the note name.
- Use aliases by adding a pipe between the note name and display text.
- Use tags sparingly for broad categories.
- Use tables for structured experiment metadata.
- Use code blocks for commands, configs, and pseudocode.

```markdown

| Field | Value |
```
